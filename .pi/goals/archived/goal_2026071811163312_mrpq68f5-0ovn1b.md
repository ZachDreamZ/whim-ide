# Goal Prompt

POLISH AND COMPLETE THE NEW AGENT CHAT EXPERIENCE

The current New Chat screen is partially implemented, but it still has functional and visual problems. Do not redesign unrelated sections. Repair the existing chat-first implementation and connect it completely to the real project, repository, conversation, and agent-run state.

The coding model does not have vision access, so use this written description as the visual reference.

CURRENT SCREEN DESCRIPTION

The application currently shows:

- A left sidebar with New chat, Scheduled, Plugins, More, Search history, Projects, Chats, Settings, and Whim workspace.
- The WHIM project appears selected and contains several entries named “continue.”
- The Chats section also contains “Agent: continue.”
- The bottom status bar says “No repository,” even though WHIM is selected.
- The main header says “New chat.”
- The empty state says “What do you want to build?”
- Four generic suggestion cards appear near the upper-middle area.
- A message composer appears separately at the bottom.
- A very large unused blank area exists between the suggestions and composer.
- There is no visible environment inspector, project-context indicator, or repository context.
- The composer contains an attachment icon, an unclear “auto” control, a placeholder, microphone icon, and send button.

MAIN PROBLEM

The screen looks like a new chat, but it is not yet a fully integrated project-aware Agent Chat. It still exposes symptoms of the old task system, does not correctly inherit the selected repository, creates low-information “continue” records, and has an unfinished empty-state layout.

==================================================
1. FIX PROJECT AND REPOSITORY CONTEXT
==================================================

The WHIM project is selected, but the bottom status bar currently says:

“No repository”

This is contradictory and must be fixed at the data-flow level.

When New Chat is opened from within a selected project:

- Inherit the selected project automatically.
- Resolve that project’s actual workspace and repository.
- Associate the new conversation with that project.
- Associate the first agent run with the same workspace and repository.
- Display the correct repository and branch in the status bar.
- Make repository tools immediately available to the agent.
- Do not require the user to select the same workspace again.

Expected example:

Project: WHIM
Repository: whim-ide
Branch: main
Workspace: C:\path\to\whim-ide

If the project genuinely has no linked repository, display a clear action:

“Connect repository”

Do not silently show “No repository” while presenting the project as ready for coding.

Trace and repair:

- Selected-project state
- Project-to-workspace mapping
- Workspace-to-repository mapping
- New-conversation initialization
- Agent-run creation
- Status-bar state
- Persistence and restoration after restart

==================================================
2. REMOVE DUPLICATE “CONTINUE” CONVERSATIONS
==================================================

The sidebar currently contains several items named:

- continue
- continue
- continue
- Agent: continue

This must be fixed in the underlying creation logic.

A continuation instruction must append to the existing conversation and run context. It must not create:

- A new project task
- A second conversation
- A new sidebar entry
- A conversation titled “continue”
- A separate “Agent: continue” record

Rules:

- One interactive user goal equals one conversation.
- Follow-up messages remain inside that conversation.
- Parent-agent continuation remains inside that conversation.
- Child-agent continuation must remain hidden from normal chat history.
- Internal orchestration messages must never become sidebar titles.
- Only an explicit New Chat action creates a new conversation.

Add idempotency protection so one message submission cannot create multiple conversations or runs.

Use meaningful title generation from the original goal.

Example:

Original request:
“Fix the inactive right-side preview pane”

Conversation title:
“Fix inactive preview pane”

Follow-up:
“continue”

The title must remain:
“Fix inactive preview pane”

Do not rename it to “continue.”

Migrate or clean existing duplicate entries safely:

- Group duplicates belonging to the same original run where possible.
- Preserve useful message history.
- Remove empty internal-only conversations.
- Do not erase valid user conversations.

==================================================
3. REBUILD THE EMPTY-CHAT LAYOUT
==================================================

The current suggestions and composer feel disconnected because a large empty region separates them.

Create a unified empty-state experience.

On a new empty conversation:

- Place the welcome title, subtitle, suggestions, and composer within one coherent centered area.
- Keep the complete group visually near the center of the available chat column.
- Do not place the suggestions near the top and the composer at the absolute bottom.
- Keep enough vertical room for smaller screens.
- Avoid excessive empty space.

Recommended structure:

New chat header

[flexible empty-state region]

What do you want to build?
Describe the task and Whim will inspect your project, make changes, and verify the result.

[suggestion cards]

[composer]

[small project-context indicator]

When the first message is submitted:

- Remove the welcome title and suggestion cards.
- Transition into the normal conversation timeline.
- Dock the composer at the bottom.
- Do not remount the whole page.
- Do not interrupt streaming.
- Preserve keyboard focus appropriately.

Reference structure:

```tsx
function ChatView() {

        </NewChatEmptyState>
      ) : (
        <>
          <ConversationTimeline />
          <StickyComposer />
        </>
      )}
    </main>
  );
}
